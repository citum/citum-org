What it does

Style-accurate output

Hundreds of journal and institutional styles supported, each tested against a shared fixture set so edge cases get caught before your manuscript does.

Formatting edge cases handled

Name disambiguation, ibid logic, delimiter variation by source type, date ranges — the details that require manual fixes in most tools are covered here by the style definition itself.

Works with sources you already have

Accepts the same reference fields used by common reference managers. No re-entry required; Citum maps existing metadata to style output.

How it works

1

Bring your references

Pass in your references as structured data — the same fields used by common reference managers.

2

Pick a style

Choose from the built-in style library or load a custom Citum YAML style. Styles declare behaviour; the engine applies it.

3

Get formatted output

Citations and bibliography rendered according to style rules — ready to copy into your document or manuscript.
