What twenty years of real use revealed

CSL was designed in 2004 without knowing the full scope of what scholars would need. A decade of styles, edge cases, and community feedback showed three gaps the data model couldn't close.

Date uncertainty gets dropped

EDTF's uncertainty markers (?, ~), open date ranges, and approximate decades map to year/month/day only. Archival and historical scholarship loses precision silently, with no error and no warning.

Styles diverge instead of extending

Adapting APA for a specific journal means forking the entire style. CSL has no inheritance model, so every house-style variant is a full copy. Changes to the parent never propagate. Maintenance compounds over time.

Multilingual names need both scripts

A citation to 村上春樹 should carry both the original script and romanization. CSL's data model has one name field per person. CSL-M added per-language layouts — but only for citeproc-js, leaving Pandoc and other processors without support.

The portability trade-off

CSL was designed implementation-independent from the start — one XML schema, many processors. That was the right call: Zotero, Pandoc, and Mendeley converged on one standard, and it worked.

But implementation-independence requires keeping the data model simple. EDTF, paired scripts, and inheritance aren't simple — so they stayed out of the spec. When the community needed more, CSL-M emerged: an extension for legal and multilingual citations, maintained by one developer for citeproc-js. Styles using CSL-M extensions don't run anywhere else.

Citum makes the opposite bet deliberately: one Rust implementation, built with twenty years of hindsight and informed by CSL styles, BibLaTeX, and the full range of community-documented edge cases. The trade-off is real — Citum runs where Citum runs. But the features are first-class, not bolted on.

CSL-M

Extended schema for legal & multilingual citations. Supported in citeproc-js and Juris-M. Styles using CSL-M extensions don't validate against the official CSL schema and won't run in Pandoc, Zotero, or other processors.

Citum

EDTF, paired scripts, and style inheritance are core to the data model and style language — not extensions. WASM, C FFI, CLI, and JSON-RPC interfaces. One implementation, done right.

How Citum handles the hard cases

EDTF dates, preserved

Uncertainty markers, open ranges, approximate decades, and seasons are stored in the reference data and rendered faithfully by the style. No silent flattening to a bare year.

Reference data (YAML)
issued: "1850/1855"  # date range
issued: "1923?"      # uncertain
issued: "1950~"      # approximate
CSL
1850  ·  1923  ·  1950
Citum
1850–1855  ·  ca. 1923  ·  ca. 1950
Child style (YAML)
extends: apa

options:
  bibliography:
    hanging-indent: false
    et-al-min: 5
Everything else inherits from APA exactly.
Pin the parent version for reproducibility.

Styles that extend, not fork

Build on any published style with scoped overrides. Pin to a specific parent version so your output doesn't change when upstream updates. Distribute your style without handing out a mystery clone of APA.

Bilingual scripts, first-class

Author names carry both the original script and a transliteration as separate fields in the data model. The style decides which to render, in what order, and how to delimit them — per locale.

Paired script fields
author:
  - original:
      family: 村上
      given: 春樹
    lang: "ja"
    transliterations:
      ja-Latn-hepburn:
        family: Murakami
        given: Haruki

Bibliographies that organize themselves

Group references by type, language, citation status, or custom criteria — without writing renderer code. Legal hierarchies, multilingual sections, and primary/secondary source divisions are expressed in the style, not patched into the document template.

For implementers: the citation pipeline

WASM, C FFI, CLI, and JSON-RPC entry points. A deterministic pipeline from structured markup to typeset output.

1

Data & Markup

References in YAML or JSON. Citation markers directly in Djot or Markdown documents. Existing BibTeX libraries convert with no loss of structured fields.

Citation markers (Djot)
[@kuhn1962]  # parenthetical
[+@kuhn1962, p. 10]  # integral
[-@kuhn1962, p. 10]  # suppress author
[@watson1953; @smith2010]  # multi-cite
Resolution trace
# Chapter 1
[+@smith2010] → Jane Doe (2010)
[@smith2010, p. 12] → (ibid., 12)
# Chapter 2 (scope reset)
[+@smith2010, p. 14] → Jane Doe (2010, 14)
2

Semantic Resolution

Disambiguation, ibid tracking, and scope-aware formatting. EDTF dates are resolved to the precision the style requests, not silently truncated.

3

Output Rendering

Semantic HTML, Typst, LaTeX, or PDF — with zero external dependencies. Output carries structured metadata for post-processing and accessibility.

Semantic HTML · APA
(<span class="csln-citation" data-ref="smith2010">Doe, <span class="csln-issued">2010</span> </span>)
